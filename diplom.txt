Дипломна робота

1. Анотація

У роботі представлено веб-платформу TeerHub, що забезпечує безпечну взаємодію між волонтерами та організаціями. Проект включає функціональні можливості реєстрації та авторизації, створення та пошуку вакансій, обмін повідомленнями та інструменти управління профілем. Ключовою задачею є побудова надійної системи зі зниженим ризиком веб-загроз, адаптивним інтерфейсом та захищеною обробкою даних.

2. Вступ

У сучасному світі цифрові платформи для волонтерської діяльності стають важливим інструментом для об’єднання людей, що бажають допомогти, з організаціями, які шукають підтримку. Однак такі системи мають особливі вимоги до безпеки та конфіденційності, оскільки обробляють персональні дані, контактну інформацію та комунікації. TeerHub проектується з акцентом на безпечний бекенд, надійний фронтенд та простий досвід користувача.

3. Мета і задачі роботи

Метою дипломної роботи є розробка функціональної платформи TeerHub з сучасною архітектурою, що забезпечує:
- реєстрацію та авторизацію користувачів;
- створення та публікацію волонтерських вакансій;
- захищений пошук і перегляд вакансій;
- обмін повідомленнями між учасниками;
- протидію загрозам веб-безпеки (XSS, ін’єкції, brute-force, захищені сесії);
- адаптивний інтерфейс для десктопу та мобільних пристроїв.

4. Огляд літератури та технологій

Для реалізації було обрано стек JavaScript, що дозволяє створювати повноцінний веб-застосунок на Node.js з Express та MongoDB. У проєкті застосовуються бібліотеки:
- Express 5 для маршрутизації запитів;
- Mongoose для моделювання даних у MongoDB;
- bcryptjs для хешування паролів;
- jsonwebtoken для аутентифікації через JWT;
- helmet, express-rate-limit та express-mongo-sanitize для підвищення безпеки.
Також використовуються стандартні web-технології HTML, CSS та JavaScript для клієнтської частини.

5. Архітектура системи

TeerHub побудований за моделлю клієнт-сервер. Серверна частина обробляє API-запити та роздає статичні сторінки, а клієнтська частина забезпечує інтерфейс для користувача.

Основні компоненти:
- Сервер (`server.js`) — конфігурація Express, підключення до MongoDB, загальна безпека, маршрутизація.
- Роутери (`routes/auth.routes.js`, `routes/vacancy.routes.js`, `routes/user.routes.js`) — логіка маршрутів для авторизації, вакансій та профілю.
- Контролери (`controllers/*.js`) — обробка бізнес-логіки.
- Моделі (`models/User.js`, `models/VolunteerVacancy.js`, `models/Message.js`) — визначення структури збережених об’єктів у базі.
- Утиліти безпеки (`utils/security.js`) — валідація, очищення, шифрування, захист від XSS та ін’єкцій.
- Статичні сторінки (`public/*.html`) та фронтенд-скрипти (`public/scripts/*.js`) — інтерфейс та взаємодія з API.

6. Реалізація функціоналу

6.1. Користувацька автентифікація

Система підтримує реєстрацію та вхід користувачів. Контролери `auth.controller.js` обробляють запити `/api/auth/login`, `/api/auth/complete`, `/api/auth/request-code` та `/api/auth/verify-code`.
Для захисту використовується `bcryptjs` для хешування паролів, rate-limit для обмеження кількості спроб входу, а також перевірка введених даних через `express-validator`.

6.2. Управління вакансіями

Користувачі можуть створювати вакансії через маршрут `/api/vacancies`. Модель `VolunteerVacancy` містить поля `title`, `description`, `location`, `contactName`, `contactEmail`, `contactPhone`, `tags`, `createdBy`, `creatorRole`, `status`.
Валідація та очищення текстових полів виконуються за допомогою утиліт з `utils/security.js`, що дозволяє уникати небезпечних символів та скриптів.

6.3. Повідомлення

Система дозволяє надсилати внутрішні повідомлення власнику вакансії через маршрут `/api/vacancies/:id/message`. Повідомлення зберігаються у модель `Message` та шифруються AES-256-GCM за допомогою `crypto`.

6.4. Інтерфейс користувача

Фронтенд реалізовано як набір статичних сторінок у папці `public`. Основні сторінки:
- `index.html` — головна сторінка;
- `login.html` та `register.html` — форми входу та реєстрації;
- `dashboard.html` — пошук і перегляд вакансій;
- `settings.html` — створення вакансії, управління профілем;
- `chat.html` — інтерфейс обміну повідомленнями.

Скрипти у `public/scripts` відповідають за взаємодію з API, рендеринг та захист від XSS на клієнті.

7. Безпека системи

7.1. Захист HTTP-заголовками

Для підвищення безпеки сервер використовує `helmet` з жорсткою політикою CSP, забороною фреймів, блокуванням змішаного контенту та захистом від MIME sniffing.

7.2. Захист від атак перебору

На маршрутах автентифікації та створення вакансій встановлено `express-rate-limit`, що обмежує кількість запитів до конкретних точок доступу.

7.3. Валідація та санітизація вводу

У модулі `utils/security.js` реалізовано функції для очищення XSS-векторів, перевірки email і пароля, очищення строкових даних та масивів. Всі введені дані проходять через ці утиліти перед збереженням до бази та обробкою.

7.4. Захист від NoSQL-ін’єкцій

Сервер має додаткову перевірку в тілі запиту на наявність підозрілих операторів Mongo ($оператори) та небезпечних шаблонів, що запобігає ін’єкціям у запити бази.

7.5. Сесії та cookies

Сесії зберігаються у MongoDB за допомогою `connect-mongo`, а cookies налаштовані як `httpOnly`, `sameSite: strict` та `secure` у виробничому середовищі.

8. Структура даних

8.1. Модель User

Модель користувача містить такі поля:
- email;
- password;
- name;
- role;
- isVerified;
- verificationCode;
- verificationExpires;
- goal;
- motivation;
- savedVacancies;
- ratings.

8.2. Модель VolunteerVacancy

Модель вакансії визначає інформацію про позицію та контактні дані, теги та власника вакансії. Поле `status` дозволяє відслідковувати відкриті та закриті оголошення.

8.3. Модель Message

Модель повідомлення зберігає зашифроване тіло, тему, одержувача та відправника.

9. Тестування та валідація

Було проведено ручне тестування основних сценаріїв:
- реєстрація і вхід користувача;
- створення вакансії та її перегляд;
- пошук і фільтрація вакансій на дашборді;
- надсилання повідомлень через API;
- перевірка коректності обробки хибних даних.

Також перевірено роботу безпеки: обмеження rate-limit, шифрування повідомлень, CSP, sanitization та обробку невірних ObjectId.

10. Висновки

Розробка TeerHub показала, що створення сучасної платформи для волонтерської взаємодії можливе з використанням Node.js та Express із власними механізмами захисту. Проєкт поєднує гнучкий інтерфейс та серверну логіку із захистом від типових веб-загроз. Подальшим розвитком може бути додавання реального чату у реальному часі, рейтингів користувачів, пошуку за місцем розташування та інтеграція з мобільними додатками.

11. Перспективи розвитку

Рекомендовано розширити функціонал TeerHub за такими напрямами:
- додати системи сповіщень та email-розсилок;
- впровадити вебсокети для миттєвого обміну повідомленнями;
- додати багатомовність інтерфейсу;
- виконати автоматизоване тестування API та UI;
- реалізувати більш складну систему рекомендацій вакансій.
